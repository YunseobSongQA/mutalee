## 1) 구조 정리 — 먼저 설계만 보여주고 내 OK 받고 진행
지금 html/css/js는 이미 분리돼 있음. 문제는 타입 분리가 아니라 파일·폴더가 너무 잘게 쪼개져
폰에서 다루기 난잡한 거다. 타입별로 더 쪼개지 마(규칙 위반). 기능 단위로 합쳐서 파일 수를 줄여라.

- reminders/(core,store,ui,generate-client) / culture/(core,ui) / notify/(notify,permission,push-client)
  → 기능 응집도 유지하면서 합칠 수 있는 만큼 합쳐라. 단, 순수함수(core)는 UI·저장소랑 절대 안 섞이게 경계 유지.
- 루트 잡동사니 정리: prompt*.txt 같은 작업메모는 앱 소스 아니니 _notes/로 빼거나 .gitignore.
  wrangler.toml이 루트+worker/ 두 군데 있는데 실제 필요한 것만 남겨라.
- data/ JSON은 그대로(데이터 분리 규칙).

제약: Vanilla JS only, 빌드 없음, 기능별 폴더 유지, 모듈 경계(계약) 안 깨기.
합칠 때 import 경로·동작 다 맞춰서 깨지는 데 없게.

→ 먼저 "파일 구조 before→after + 뭐가 어디로 합쳐지는지"만 제안. 코드는 OK 받고.

## 2) 알림 중복 버그 — 원인 분석부터
증상: 잠금화면에 같은 노티('주식 확인')가 동시에 2번 뜬다.
의심 지점:
- 로컬 포그라운드 알림(notify.js의 loadNotifiedToday dedup)과 서버 웹푸시(worker/cron.js의
  record.notifiedToday dedup)가 저장소가 서로 달라 같은 노티를 각자 한 번씩 쏘는 중복.
- 또는 functions/api/sync.js가 endpoint 해시로 키를 잡는데, iOS에서 구독 endpoint가 갱신되면
  KV에 같은 기기 구독이 두 개 남아 cron이 둘 다 발송.
실제 원인 어느 쪽인지 코드로 짚고, "앱 열려 있을 땐 로컬·푸시 중 하나만 + 죽은/중복 구독 정리"로 고쳐라.
원인 진단 → 고칠 지점 → (승인 후) 변경분만.

공통: 한글로. 막혀도 묻지 말고 가정해서 끝까지, 가정은 끝에 정리.
